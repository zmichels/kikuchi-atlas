# Kikuchi Lab Incubator

The incubator preserves worthwhile directions without silently adding them to
the exceptional-forsterite milestone. A record is evidence-aware planning, not
an implementation promise, accepted architecture, or hidden acceptance
criterion.

Kikuchi Lab remains a companion project now: it uses upstream scientific
engines behind project-owned contracts while retaining recipes, provenance,
diagnostics, and decisions locally. An independent engine is a possible later
project only after the incubated evidence supports that boundary.

## Record contract

Every record states its motivation, evidence already present in this
repository, dependencies, unresolved questions, linked decisions or
experiments, exactly one promotion trigger, and explicit present non-goals.
Promotion requires a new or updated repo-native work item and, when the change
affects architecture or scientific interpretation, a reviewed decision record.
Until then, incubator work must not change the scope or acceptance state of
[KIKU-F001](../work/KIKU-F001.md).

## Present boundary

These records do not authorize new datasets, forks, upstream modifications,
ebsdx integration, 3D viewing or print exports, additional phase deliverables,
or an independent simulator. The active milestone remains one accepted,
reproducible forsterite pattern through the workflow defined in the
[approved design](../superpowers/specs/2026-07-12-kikuchi-companion-design.md).
